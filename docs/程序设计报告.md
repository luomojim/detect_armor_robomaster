## 第二题：装甲板位姿解算系统

### 系统整体流程设计

```
第二题主流程：
开始 → 导入视频 → 初始化卡尔曼滤波 → 读取帧
    ↓
图像预处理（通道分离、阈值处理）
    ↓
轮廓检测 → 灯条检测 → 灯条匹配
    ↓
角点提取 → PnP解算
    ↓
计算yaw角 → 卡尔曼滤波
    ↓
计算三维旋转中心 → 重投影
    ↓
可视化绘制
    ↓
是否有下一帧？ → 是 → 返回读取帧
    ↓ 否
结束
```

#### 装甲板角点检测阶段

在预处理后的二值图像上进行轮廓检测，通过armorTracker类实现：
- 检测所有轮廓，提取灯条候选区域
- 对每个轮廓计算最小外接旋转矩形
- 根据长宽比、面积等特征筛选灯条
- 从左往右排序，从上往下排序
- 匹配左右灯条，形成装甲板对
- 计算装甲板的四个角点坐标

#### 位姿解算阶段

使用PnP算法解算装甲板的位姿：
- 定义装甲板的世界坐标系（四个角点的3D坐标）
- 使用solvePnP函数求解旋转向量和平移向量
- 从旋转向量计算偏航角和俯仰角
- 计算装甲板到相机的距离

#### 卡尔曼滤波处理阶段

对yaw角进行卡尔曼滤波：
- 计算yaw角的变化速度
- 使用卡尔曼滤波平滑yaw角速度
- 预测未来的yaw角变化
- 更新滤波后的yaw角

### 函数结构组织

#### 图像预处理类（openCVBasicFX）

**功能**：提供图像预处理功能、阈值处理操作。

**核心方法**：
- `split_image_channels(Mat& image)`：分离图像通道
- `to_Threshold_Image(Mat& image, int threshold_low, int threshold_high)`：阈值处理

#### 3.2 角点检测类（armorTracker）

**功能**：实现装甲板检测、灯条匹配、角点计算等功能。

**核心方法**：
- `detect_armors(vector<vector<Point>> contours)`：从轮廓中检测灯条
- `match_armor()`：匹配左右灯条，形成装甲板
- `solve_pnp()`：使用PnP算法解算位姿
- `find_robot_center()`：计算三维旋转中心
- `caculate_delta_angle(double delta_t)`：计算角度变化
- `reset()`：重置跟踪状态
- `draw_result(Mat &frame, Mat &ans)`：绘制结果

**成员变量**：
- `vector<RotatedRect> light_bars`：检测到的灯条
- `armor armors`：装甲板信息
- `bool found`：是否找到装甲板
- `Mat camera_matrix`：相机内参矩阵
- `Mat camera_fix`：相机畸变参数
- `double yaw, pitch, distance`：位姿参数
- `Mat translation_vector, rotation_vector`：PnP解算结果
- `vector<Point3f> center_3d`：三维旋转中心
- `vector<Point2f> center_2d`：二维旋转中心
- `double last_yaw`：上一帧yaw角
- `bool is_tracking`：是否连续跟踪

#### 位姿解算类（集成在armorTracker中）

**功能**：使用PnP算法解算装甲板的位姿。

**核心方法**：
- `solve_pnp()`：调用OpenCV的solvePnP函数
- 从旋转向量计算偏航角，俯仰角，中心离相机光心距离

**实现原理**：
- 定义装甲板的世界坐标系（四个角点的3D坐标）
- 使用检测到的2D角点坐标
- 调用solvePnP求解旋转向量和平移向量
- 用atan2来计算偏航角和偏航角

#### 三维旋转中心计算类（集成在armorTracker中）

**功能**：计算机器人的三维旋转中心，并投影到二维图像。

**核心方法**：
- `find_robot_center()`：计算三维旋转中心
- 使用重投影将3D中心投影到2D图像

**实现原理**：
- 假设机器人旋转中心位于装甲板后方一定距离（robot_distance=350mm）
- 根据装甲板的位姿（旋转矩阵和平移向量）计算旋转中心的3D坐标
- 将3D中心重投影到2D图像平面

### 功能模块实现详解

#### 位姿解算

**PnP算法或其他位姿解算方法选择与实现**：
- 使用OpenCV的solvePnP函数
- 输入：3D世界坐标点（装甲板四个角点的已知尺寸）、2D图像坐标点
- 输出：旋转向量、平移向量

**yaw角、pitch角计算原理**：
- yaw角 = atan2(x,z),前和右构成偏航角
- pitch角 = atan2(y,z),前和下构成的俯仰角

**距离(distance)解算方法**：
- 用cv库里面的范数，来求空间两个点距离
- 表示装甲板中心到相机光心的距离

**结果可视化实现**：
- 在图像上显示yaw、pitch、distance数值
- 绘制yaw角随时间变化的曲线
- 使用不同颜色区分原始值和滤波值

#### 整车旋转中心标注

**三维旋转中心计算方法**：
- 假设机器人旋转中心位于装甲板法向量方向，距离装甲板中心robot_distance
- 计算装甲板的法向量：从旋转矩阵提取第三列
- 旋转中心3D坐标 = 装甲板中心 + 法向量 * robot_distance

**从三维到二维图像的重投影原理**：
- 使用相机投影模型：x = K * [R|t] * X
- 其中K为相机内参矩阵，[R|t]为外参矩阵，X为3D点
- 使用projectPoints函数进行重投影

**旋转中心在图像中的标注实现**：
- 将3D旋转中心重投影到2D图像
- 使用黄色圆点标注旋转中心
- 显示旋转中心坐标文本

#### 卡尔曼滤波处理

**针对yaw角的卡尔曼滤波模型设计**：
- 状态向量：$x = [yaw, yawspeed]^T$
- 状态方程：$x_k = F * x_{k-1} + w_k$
- 观测方程：$z_k = H * x_k + v_k$
- $F = [[1, Δt], [0, 1]]$
- $H = [1, 0]$

**滤波算法实现细节**：
1. 预测：$x̂_k^- = F * x̂_{k-1}$
2. 预测协方差：$P_k^- = F * P_{k-1} * F^T + Q$
3. 计算卡尔曼增益：$K_k = P_k^- * H^T * (H * P_k^- * H^T + R)^{-1}$
4. 更新状态：$x̂_k = x̂_k^- + K_k * (z_k - H * x̂_k^-)$
5. 更新协方差：$P_k = (I - K_k * H) * P_k^-$

### 遇到的难点及解决思路

#### 三维旋转中心计算精度问题

**难点**：这里一开始想用最小二乘拟合，but，我数学太烂了，根本想不到怎么在一堆法向量里面拟合出一个距离他们都很小的点，所以这个方案最后被我否决了，但是正确的思路一定是这样的

**解决思路**：我用了一个很偷鸡的方法,看原视频可以推出装甲板和旋转中心大概是一个直角三角形的关系，于是乎只要知道整个旋转半径就可以用勾股定理反推出旋转中心，但是旋转半径是不知道的，只能去假设，这也是现在这个解决方法的局限性。
通过查阅RoboMaster 2026 机甲大师高校系列赛机器人制作规范手册V1.2.0（20251230）里面的步兵机器人章节，可以看到机器人的地盘尺寸最大伸展尺寸（mm） 在任意情况下，机器人的长、宽、高不得大于800*800*800。这里取350mm作为半径。就可以推出一个相对合理的旋转中心。
